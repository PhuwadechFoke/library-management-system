# เอกสารขอบเขตโครงงาน (Project Scope)
## ระบบบริหารจัดการห้องสมุด (Library Management System)

เอกสารนี้สรุปขอบเขตโครงงานการพัฒนาระบบบริหารจัดการห้องสมุด (Library Management System) ซึ่งเป็นเว็บแอปพลิเคชันแบบ Full-Stack พัฒนาด้วยเฟรมเวิร์ก **Next.js (App Router)** ร่วมกับ **Prisma ORM** และฐานข้อมูล **PostgreSQL** โดยแยกโครงสร้างขอบเขตงานออกเป็นส่วนของ **Frontend (ส่วนติดต่อผู้ใช้)** และ **Backend (ส่วนประมวลผลหลังบ้าน)** ดังรายละเอียดต่อไปนี้

---

## 1. ข้อมูลทั่วไปของโครงงาน (Project Overview)
* **ชื่อโครงการ:** ระบบบริหารจัดการห้องสมุด (Library Management System)
* **รูปแบบการพัฒนา:** Monorepo (Next.js 14 App Router) ซึ่งผสาน Frontend และ Backend เข้าไว้ด้วยกันอย่างเป็นระบบ
* **สถาปัตยกรรม:** Client-Server Architecture โดยใช้ Server Actions และ API Routes ของ Next.js ในการประมวลผลร่วมกับฐานข้อมูล

---

## 2. ขอบเขตการทำงานส่วนหน้าบ้าน (Frontend Scope)
ส่วนติดต่อผู้ใช้งานที่แสดงผลบนเว็บเบราว์เซอร์ ได้รับการออกแบบให้รองรับ Responsive Web Design (ทำงานได้ดีทั้งคอมพิวเตอร์ แท็บเล็ต และสมาร์ทโฟน) โดยแบ่งขอบเขตตามบทบาทผู้ใช้งาน (Role-Based Access Control) ดังนี้:

### 2.1 ส่วนของผู้ใช้ทั่วไป / สมาชิก (User Interface)
1. **ระบบสมัครสมาชิกและยืนยันตัวตน (Registration & Authentication):**
   * หน้าสมัครสมาชิก (Register) และเข้าสู่ระบบ (Login)
   * ระบบกู้คืนรหัสผ่าน (Forgot Password & Reset Password)
   * หน้าลงทะเบียนข้อมูลส่วนตัวเบื้องต้น (Onboarding) เมื่อเข้าสู่ระบบเป็นครั้งแรก เพื่อบันทึกข้อมูล เช่น คำนำหน้าชื่อ, ชื่อ-นามสกุลจริง, รหัสนักศึกษา/พนักงาน, เบอร์โทรศัพท์, ระดับการศึกษา และชั้นปี
2. **หน้าหลักและการค้นหา (Home & Discovery):**
   * ส่วนแสดงแบนเนอร์ประชาสัมพันธ์ (Banner Slider) แบบสไลด์รูปภาพที่ดึงข้อมูลสดจากฐานข้อมูล
   * ระบบคัดกรองหนังสือตามหมวดหมู่ (Category Filter)
   * ระบบค้นหาหนังสือ (Book Search Dialog) แบบพิมพ์ค้นหาแบบ Interactive
3. **การเข้าถึงรายละเอียดของหนังสือ (Book Details):**
   * แสดงรายละเอียดข้อมูลหนังสือ เช่น ชื่อผู้แต่ง, ราคา, จำนวนคงเหลือในคลัง, รูปหน้าปก และรายละเอียดเนื้อหาโดยย่อ
   * ระบบแสดงผลรายการหนังสือที่คล้ายกันหรือแนะนำ (Swiper Carousel)
4. **ระบบรายการโปรดและจองหนังสือ (Favorites & Reservations):**
   * ปุ่มบันทึกหนังสือเล่มโปรด (Favorite Book) เพื่อเก็บไว้ในโปรไฟล์ส่วนตัว
   * ปุ่มทำรายการจองหนังสือ (Reserve Book) โดยผู้ใช้สามารถจองหนังสือผ่านหน้าเว็บเพื่อกำหนดสิทธิ์การยืมก่อนเข้าไปรับหนังสือจริง
5. **การจัดการบัญชีผู้ใช้งาน (User Profile & Account Settings):**
   * หน้าการตั้งค่าบัญชี (Settings) สำหรับแก้ไขข้อมูลส่วนตัว อัปเดตรูปภาพโปรไฟล์ (ผ่านบริการอัปโหลดภาพ)
   * ตรวจสอบประวัติการยืม-คืน และประวัติการจองของตนเอง

### 2.2 ส่วนของผู้ดูแลระบบ / บรรณารักษ์ (Admin / Librarian Dashboard)
1. **แดชบอร์ดสรุปข้อมูล (Admin Analytics Dashboard):**
   * แสดงตัวเลขทางสถิติรวมของระบบ เช่น จำนวนหนังสือทั้งหมด, จำนวนสมาชิกทั้งหมด, ยอดการยืมหนังสือในปัจจุบัน และยอดการจองที่รออนุมัติ
   * กราฟและสถิติข้อมูลกิจกรรมการยืม-คืน
2. **การจัดการข้อมูลระบบ (Content Management System - CRUD):**
   * **ระบบจัดการหนังสือ (Book Management):** เพิ่ม, แก้ไข, ปรับปรุงรูปปก, ลบข้อมูลหนังสือ และกำหนดสถานะการเปิดใช้งาน (Active/Inactive)
   * **ระบบจัดการหมวดหมู่ (Category Management):** เพิ่มและแก้ไขหมวดหมู่หนังสือเพื่อจัดกลุ่ม
   * **ระบบจัดการแบนเนอร์ (Banner Management):** เพิ่ม ลบ แก้ไขสไลด์ภาพประชาสัมพันธ์บนหน้าแรก
   * **ระบบจัดการหนังสือเฉพาะแผนก (Department Book Management):** บริหารจัดการทรัพยากรหนังสือเรียนเฉพาะกลุ่มวิชา/สาขาวิชา
3. **ระบบควบคุมกิจกรรมยืม-คืน (Circulation & Operation Management):**
   * **การอนุมัติการยืม (Borrow Approval):** บรรณารักษ์บันทึกและอนุมัติการยืมหนังสือของสมาชิก โดยระบุผู้ยืม, หนังสือ, วันที่ยืม, และวันครบกำหนดส่งคืน
   * **การอนุมัติการคืน (Return Approval):** บันทึกการรับคืนหนังสือ ตรวจสอบสถานะการชำรุดเสียหาย และคำนวณค่าปรับอัตโนมัติ (Fine) หากคืนหนังสือเกินกำหนดเวลาส่งคืน
   * **การตรวจสอบการจอง (Reservation Management):** อนุมัติหรือยกเลิกคิวจองหนังสือของสมาชิก
4. **การบริหารสิทธิ์ผู้ใช้และประวัติระบบ (User Management & Audit Logs):**
   * จัดการข้อมูลสมาชิกและแก้ไขสถานะบัญชี (User Management)
   * จัดการรายชื่อบรรณารักษ์และผู้ดูแลระบบ (Admin Role Assignment)
   * หน้าตรวจสอบประวัติกิจกรรมที่เกิดขึ้นในระบบ (System Activity Logs) เพื่อความโปร่งใส เช่น บันทึกว่าแอดมินคนใดเป็นผู้เพิ่มหนังสือเล่มใหม่ หรืออนุมัติการยืมเล่มใด

## 3. ขอบเขตการทำงานส่วนหลังบ้าน (Backend Scope)
ส่วนประมวลผลเบื้องหลัง การจัดการเซสชัน ความปลอดภัย และระบบฐานข้อมูล ประกอบด้วย:

### 3.1 สถาปัตยกรรมเซิร์ฟเวอร์และการประมวลผล (Application & Server Logic)
1. **Next.js App Router (Server-side Runtime):**
   * ใช้ Server Components ในการดึงข้อมูลโดยตรงจากฐานข้อมูลเพื่อความรวดเร็วและความปลอดภัย
   * ใช้ Server Actions ในการบันทึกข้อมูล (CUD Operations) ป้องกันการรั่วไหลของข้อมูลรับเข้า (Input Validation)
   * บริการ API Routes (`/api/*`) สำหรับการจัดส่งข้อมูลภายนอกและจัดการ Authentication
2. **ระบบสิทธิ์และความปลอดภัย (Authentication & Session Management):**
   * บูรณาการกับ **NextAuth.js** ในการลงชื่อเข้าใช้งาน
   * ป้องกันรหัสผ่านรั่วไหลด้วยการเข้ารหัสแบบแฮชชิ่ง (Password Hashing) โดยใช้ไลบรารี `bcrypt`
   * การตรวจสอบสิทธิ์การเข้าถึงข้อมูลตามบทบาท (Role-Based Authorization Middleware) เพื่อป้องกันการเข้าถึง Dashboard จากผู้ใช้ระดับปกติ
3. **ระบบฐานข้อมูลและการจัดการ (ORM & Database Layer):**
   * ใช้ **Prisma Client** ในการเข้าถึงฐานข้อมูลในรูปแบบ Type-safe ORM
   * ใช้ฐานข้อมูลเชิงสัมพันธ์ **PostgreSQL** ในการจัดเก็บข้อมูลทั้งหมดของระบบ
   * ระบบการโยกย้ายฐานข้อมูล (Database Migrations) เพื่อประยุกต์ใช้โครงสร้าง Schema

### 3.2 ระบบบริการและฟังก์ชันเสริม (Integration Services)
1. **ระบบจัดเก็บไฟล์สื่อ (Cloud File Storage Integration):**
   * บูรณาการกับ **UploadThing API** เพื่อให้บริการฝากไฟล์รูปภาพความเร็วสูง เช่น ภาพโปรไฟล์ผู้ใช้ และภาพปกหนังสือ
   * มีระบบลบรูปเก่าออกจาก Cloud Storage เมื่อมีการอัปเดตหรือลบรายการ เพื่อป้องกันการสิ้นเปลืองพื้นที่จัดเก็บ
2. **ระบบอีเมลบริการ (Email Notification Service):**
   * บูรณาการกับ **Nodemailer** หรือ **Resend API** ส่งเมลยืนยันการสมัครสมาชิก (Verify Email)
   * ระบบส่งลิงก์สำหรับเปลี่ยนรหัสผ่านใหม่ (Reset Password Token) ที่มีระบบจำกัดเวลาใช้งาน (Expired Token) เพื่อความปลอดภัย
3. **การคำนวณวันและค่าปรับอัตโนมัติ (Business Logic Rules):**
   * คำนวณวันกำหนดส่งคืนจากสูตรเวลาปัจจุบันเทียบกับระยะเวลาที่บรรณารักษ์ระบุ
   * ตรวจวัดประวัติการคืนหนังสือเพื่อปรับสถานะเป็น `OVERDUE` (เกินกำหนด) หรือ `RETURNED` (คืนแล้ว) แบบอัตโนมัติ และคิดคำนวณค่าปรับสะสมจากวันที่เกินกำหนด

## 4. โครงสร้างข้อมูลระบบ (Database Entity-Relationship Summary)
ระบบเก็บข้อมูลในรูปแบบฐานข้อมูลเชิงสัมพันธ์ (Relational Database) มีตารางข้อมูลที่เกี่ยวเนื่องกัน 10 ตารางดังนี้:

* **User**: ข้อมูลบัญชีผู้เข้าใช้งานเบื้องต้น (อีเมล, รหัสผ่านเข้ารหัส, บทบาทของไอดี)
* **UserProfile**: ข้อมูลรายละเอียดส่วนตัวของผู้ใช้งาน เชื่อมโยงแบบ 1:1 กับบัญชี User
* **Book**: รายการหนังสือทั้งหมดในระบบห้องสมุด (ราคา, ผู้แต่ง, คลังคงเหลือ)
* **Category**: หมวดหมู่หลักของหนังสือเพื่อแยกประเภท
* **DepartmentBook**: รายการหนังสือเรียนแยกตามแผนกหรือระดับการศึกษา
* **FavoriteBook**: ตารางเก็บความสัมพันธ์ระหว่างหนังสือที่ผู้ใช้ชื่นชอบ
* **Reservation**: คิวจองหนังสือของผู้ใช้ที่ประสงค์จะมายืมหนังสือในอนาคต
* **Borrow**: ข้อมูลประวัติการทำรายการยืม-คืนหนังสือ, ค่าปรับ และสถานะการยืม
* **Banner**: รูปภาพโฆษณา/ประชาสัมพันธ์หน้าเว็บ
* **Activity**: บันทึกประวัติกิจกรรมการทำงานของผู้ดูแลระบบภายในระบบ

## 5. เทคโนโลยีหลักที่ใช้พัฒนา (Technology Stack)

### 5.1 ด้าน Frontend (Client-side)
* **Framework:** React 18 & Next.js 14 (App Router)
* **Language:** JavaScript (ES6+)
* **Styling (CSS):** Tailwind CSS, DaisyUI
* **UI Components:** Radix UI (Shadcn UI component base), Swiper (สำหรับ Carousel ภาพ), Framer Motion (สำหรับแอนิเมชันตอนเปิดปิดหน้าจอ), SweetAlert2 (สำหรับแสดง Dialog โต้ตอบ)
* **State Management & Querying:** React Redux / `@reduxjs/toolkit` ร่วมกับ `@tanstack/react-query` สำหรับการสืบค้นข้อมูล

### 5.2 ด้าน Backend & Database (Server-side)
* **Backend Framework:** Next.js Server Actions & API Routes
* **Database ORM:** Prisma Client (`@prisma/client` & `prisma`)
* **Database Engine:** PostgreSQL
* **Authentication:** NextAuth.js & `bcrypt`
* **File Upload Service:** UploadThing
* **Mail Dispatcher:** Nodemailer / Resend / `@react-email/components`
